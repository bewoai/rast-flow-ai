## 3. AE-MOGRT atılımı (en kritik bölüm)

### Sorun neydi
Premiere ExtendScript API'si **yalnızca After Effects'te yapılmış** MOGRT'ların metnine erişebiliyor. Bizim eski `caption.mogrt` **Premiere'de** yapılmıştı → `clip.getMGTComponent()` hep `null` dönüyordu → metin doldurulamıyordu. (AutoCut/FireCut kaynak kodu `.jsxbin` = derlenmiş, okunamadı; teknik Adobe'un public API'sinden öğrenildi.)

### Çözüm
1. **AE'de caption şablonu** yapıldı (`altyazimogrt.aep`): Text katmanı + auto-size Rectangle (box) + pop-in animasyon (Scale/Opacity keyframe).
2. **Box auto-size** = Rectangle Path Size/Position expression'ı `sourceRectAtTime()` ile → kutu metnin uzunluğuna göre otomatik genişler.
3. **Source Text** Essential Graphics paneline sürüklendi (param adı "Altyazı"); ayrıca Background Color/Opacity/Yuvarlaklık/Stroke paramları açıldı.
4. **Metin doldurma (kodda):** `getValue()` JSON string'ini parse et → `textEditValue = metin` + **`fontTextRunLength = [metin.uzunluğu]`** (kritik, bu olmadan uygulanmıyor) + **tüm `*RunLength` dizileri** `[uzunluk]`'a (font/renk/STROKE biçimi yeni metne taşınır).
5. **Şablon yerleştirildi:** Yeni AE export `templates/caption.mogrt`'a kopyalandı (eski Premiere-yapımı `.bak` olarak yedeklendi).

## 4. AÇIK İŞLER / sıradaki adımlar

1. **MOGRT'u TEST ET** (öncelik): Paneli kapat-aç → **✨ Animasyonlu Altyazı**'ya bas.
   - Beklenen: Günlük'te `metin doldurulan: N` (N > 0 ise çalışıyor 🎯) + timeline'da düzenlenebilir/animasyonlu/auto-box altyazılar.
   - Olmazsa Günlük'teki `MOGRT teşhis:` / `MOGRT paramlar:` satırlarına bak.

2. **Panel stil kontrollerini MOGRT paramlarına bağla** (tam panel-stili için):
   - Test sonrası Günlük'te **`MOGRT paramlar: 0:'Altyazı'=... 1:'Background Color'=... 2:'Background Opacity'=...`** dökümü çıkıyor (`_dumpMogrtParams`).
   - Bu döküm her paramın **setValue formatını** verir (renk dizisi mi 0-1, sayı mı).
   - Sonra paneldeki Kutu Rengi/Opaklık/Yuvarlaklık/Stroke kontrolleri bu exposed paramlara `setValue` ile bağlanır → "tüm stili panelden değiştir" tam çalışır.

3. **Şablon export'u:** Bundan sonra AE'den **doğrudan** `templates/caption.mogrt`'a export et (root'a `captions.mogrt` değil — kod `templates/`'i okur).

4. Düzenlenebilir caption track (`createCaptionTrack`) Premiere'de doğrulanmadı — alternatif yol olarak test edilebilir.

## 5. Teknik notlar / öğrenilen dersler

- **AE-MOGRT zorunlu:** Premiere'de yapılan MOGRT'ın metnine ExtendScript erişemez.
- **Metin doldurma:** `textEditValue` + `fontTextRunLength=[uzunluk]` + tüm `*RunLength` dizileri → biçim (font/renk/stroke) korunur.
- **Box auto-size:** Rectangle Path Size/Position'a `sourceRectAtTime()` expression'ı; metin değişince kutu yeniden boyutlanır.
- **FFmpeg:** `lib/ffmpeg/ffmpeg.exe` (gitignore: `*.exe` → repoya girmez); `-ss` `-i`'den ÖNCE = seek, büyük dosyada boyut sınırı yok.
- **importFiles asenkron:** import + yerleştirme AYRI evalScript çağrısı + ~1.5 sn gecikme (aynı çağrıda "öğe bulunamadı").
- **`getSystemPath(EXTENSION)`** + `window.location` fallback → ffmpeg/mogrt yolu güvenilir bulunur.
- **Junction:** `%APPDATA%\Adobe\CEP\extensions\com.pr.workflow` → DEV'e bağlı (doğrulandı). Junction'ı `Remove-Item -Recurse` ile SİLME — hedefi silebilir.
- **DEV/RUN:** Junction sayesinde değişiklikler canlı; client (app.js/html/css) için ↻ yeterli, **JSX değişikliğinde panel kapat-aç** gerekir.
